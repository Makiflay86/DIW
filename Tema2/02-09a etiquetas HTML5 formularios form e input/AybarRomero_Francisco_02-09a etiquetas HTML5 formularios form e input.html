2º Escribe en un documento html con el titulo <h1> GET/POST </h1> y con tus palabras en que consiste el paso de datos mediante GET y post y las ventajas de cada uno

GET/POST

Es la forma que se envían los datos al servidor, el get envía los datos en la URL mientras el post lo hace en el cuerpo de la solicitud HTTP.


Ventajas

  • GET almacena en cache los datos, pero todo el mundo lo puede ver.
  • POST los datos que se envían no es visible ya que se envía por HTTP.

    Atributos

    • accept-charset: Codificación de caracteres que se utilizará para pasar los contenidos del formulario.
    • autocomplete: El navegador automáticamente completará los datos del formulario basándose en los valores para esos mismos controles que el usuario haya rellenado en otros formularios.
    • enctype: Solo es válido para el método POST. Indica el formato en el que los datos del formulario son enviados.
    • novalidate: No validara datos.
    • target: Muestra la respuesta del formulario dependiendo de como termina, si pone _blank se abrirá en otra pestaña.

    6º Escribe en un documento htm con el titulo <h1> Atributos button</h1> una lista de ordenada que explique cada atributo.

    Atributos button

    1. Formaction: Especifica la URL a la que se envían los datos del formulario al hacer clic en el botón, anulando el atributo "action" del elemento "<form>" padre.
    2. Formethod: Define el método HTTP (como "GET" o "POST") a utilizar para enviar los datos del formulario, anulando el atributo "method" del elemento "<form>" padre.
    3. Formenctype: Determina cómo se deben codificar los datos del formulario antes de enviarlos al servidor (por ejemplo, "application/x-www-form-urlencoded" o "multipart/form-data"), anulando el atributo "enctype" del elemento "<form>" padre, y solo es relevante para el método "POST".
    4. Formnovalidate: Es un atributo booleano que, si está presente, indica que los datos del formulario no deben validarse al ser enviados, anulando la validación del elemento "<form>" padre.
    5. Formtarget: Especifica el contexto de navegación (una ventana, pestaña o iframe) donde se mostrará la respuesta del servidor después de enviar el formulario (valores comunes son "_self", "_blank", "_parent", "_top"), anulando el atributo "target" del elemento "<form>" padre.
